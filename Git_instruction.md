# **2. Проверим СТАТУС** 

После создания репозитория **.git** в папке **Git_Test** необходимо убедится, что всё в порядке и мы точно находимся в нужной папке, проверим точно что репозиторий появился т.е. узнать СТАТУС.
Пишем:

**git status**

Чтобы создать ветки, пишем:

Например, создадим 4 ветки. Пусть будут такие:

2. ветка для уточнений\дополнений параграфа №1 (**paragraph_1**)

**git branch images**

**git branch paragraph_1**

**git branch paper**

**git branch question**

Чтобы убедиться, что все намеченные ветки созданы, пишем 

**git branch**

и видим список наших веток.

Теперь переходим на любую ветку, например нам надо на ветку *images* (чтобы перейти на ветку надо написать git checkout и дальше название ветки), пишем

> # **Слияние веток** Команда git merge

Чтобы одна ветка вошла в другую, надо войти **В** ветку **В** которую будет вливаться другая. Например, у нас есть ветка **012**  и ветка **Z**
Ели нам надо чтобы в **012** вошла **Z**, то входим в ветку **012** и вносим ветку **Z** и если надо, то можно наоборот.  (в ветку Z внести ветку 012)
Пример

вошли в ветку **012** (команда следующая)

**git checkout 012**

вливаем ветку **Z** (команда следующая)

**git merge Z**

Все! Теперь обе ветки стали единым целым документом
